4 jasper_field_doctype Data Field Doctype Value

Put here the field of the doctype you want associate this parameter value with. The doctype is the doctype selected in report creation and docname is the doc where you call the report. OR, You can also use the Parameter value box to set the role. Like this: Put doctype:docname:docfield in Parameter value to associate the parameter value with docfield of doctype and docname. If you don't want hardcode the value you can pass this in params argument in jasper.get_jasper_report function. This role is for use as in the following example. Example: jasper.get_jasper_report({jr_name: "Cherry Local", jr_format: "pdf", doctype_type:"Form", params: {language: "System Settings:language"}, doctype:"User", docids:["luisfmfernandes@gmail.com"]}). In this way you are getting in the System Settings doctype the field language of the system to use as the value for this parameter and show him in the Cherry Local report that has information on the user luisfmfernandes@gmail.com.

6 is_copy Select Type

1) Is for copies: To set Original, Duplicated or Triplicate in the document. 2) Is for where clause: Is for SQL selects of type:"select x,y from $P!{param}". The System will create the where clause for the document. 3) Is doctype id: The value(s) of this parameter will be used in SQL select, but the system will not create a where clause. The where clause already exist in Jasper Report. 4) Is for server hook: The system will ask for the value(s) of the parameter on the server side. This could be the whole sql if in "queryString tag" of Jasper Reports only is $P!{param}. 5) If you select "Other" you must give the value or it will be ignored.
